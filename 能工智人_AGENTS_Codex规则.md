# AGENTS.md - Codex 工作规则

本文件用于约束 Codex / AI 编程助手在《能工智人》仓库中的工作方式。

---

## 项目定位

《能工智人》是 Unity 2D 单机 Roguelike Demo，参赛方向是 AI 叙事剧情游戏。核心不是普通地牢战斗，而是：

1. 装备武器流：玩家亲自战斗、强化自身。
2. 机械智斗流：玩家制作并部署机械装置。
3. AI 漫画导演：根据玩家行为生成 Boss 台词、工坊建议、结算战报。
4. 手绘漫画 UI：主菜单、背包、机械蓝图、Boss 登场、结算战报都要有漫画/蓝图视觉语言。

---

## 工作原则

1. 先做 MVP，不做大而全。
2. 每次只实现一个小功能，不要跨模块大重构。
3. 代码要能被 Unity 编译。
4. 不要导入第三方插件，除非用户明确批准。
5. 不要使用商业游戏素材、音乐、字体或数值。
6. 不要照搬《死亡细胞》或其他商业游戏数值。
7. 不要复制无许可证 GitHub 仓库代码。
8. 参考开源项目时，必须在 `docs/OPEN_SOURCE_AUDIT.md` 记录。
9. AI 功能先用 Mock 实现，真实 API 作为可选增强。
10. 演示稳定性优先于功能数量。

## 命名规范

- 命名空间：`NengongZhiren`
- 类名：PascalCase
- 私有字段：`camelCase`
- Inspector 字段：`[SerializeField] private ...`
- 数据类后缀：`Data`
- UI 控制类后缀：`UI`
- 服务接口前缀：`I`

示例：

```csharp
namespace NengongZhiren
{
    public class PlayerController : MonoBehaviour
    {
        [SerializeField, Tooltip("玩家移动速度")]
        private float moveSpeed = 5f;
    }
}
```

## 架构约束

### Core

负责游戏状态、统计数据、流程切换。

### Player

负责移动、战斗、背包、属性。

### Enemy

负责普通怪物和 Boss 行为。

### Machines

负责机械数据、制作、部署、伤害、击杀统计。

### AI

负责叙事接口、Mock 文案、Prompt 构造、安全过滤。

### UI

负责主菜单、HUD、背包、蓝图、Boss 漫画登场、结算漫画。

## 禁止事项

- 禁止一次性生成 30 个脚本但互相无法编译。
- 禁止把所有逻辑塞进 `GameManager`。
- 禁止写无法在 Unity Inspector 配置的硬编码引用。
- 禁止使用 `FindObjectOfType` 作为主要依赖查找方式，除非 MVP 临时兜底。
- 禁止让 AI 直接决定实时战斗全部行为。
- 禁止没有回退策略地调用外部 AI API。
- 禁止生成夸大项目能力的 README 描述。

---

## 优先级

P0：必须完成

- Unity 可编译
- 玩家移动与攻击
- 敌人追击与死亡
- 掉落/宝箱/材料
- 至少一种机械能部署并造成伤害
- Boss 能登场并被击败
- 结算页能根据统计显示不同文本

P1：路演需要

- 3 种机械
- 3 种怪物
- Boss 台词随玩家风格变化
- 手绘漫画风 UI
- 漫画战报

P2：冲奖增强

- 真实 AI API
- 多关卡
- 隐藏事件
- 更多装备词条
- 更完整视觉资产

## 代码审查关注点

- Unity 编译错误
- 空引用
- Inspector 未绑定风险
- Update 中频繁查找对象
- 类职责过大
- 没有事件解绑
- 数据硬编码过多
- AI 文案不可控
- 开源来源未记录
